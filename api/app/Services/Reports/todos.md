# Implement Business Reports Module

Create a Reports module under:

app/Http/Controllers/Reports

Use Services where necessary.

All reports must support the following filters:

- today
- last_7_days
- last_30_days
- this_month
- this_year
- last_year

Use Carbon for date filtering.

The authenticated user's business_branch_id should be used to scope all branch-level reports unless otherwise specified.

Return JSON responses.

# Branch Performance Report

Controller:
BranchPerformanceReportController

Service:
BranchPerformanceReportService

Primary Data Source:
CashFlow

Purpose:
Compare business branches based on financial performance.

Reason:
CashFlow already records all revenue and expenses generated by each branch. Using CashFlow avoids recalculating revenue from Sales and provides a centralized source of truth for financial reporting.

Filters:

- today
- last_7_days
- this_month
- this_year

Apply date filtering using:

cash_flows.transaction_date

Only include:

status = 'completed'

## Revenue Metrics

Revenue should be calculated from:

type = 'sale'

or

category = 'product_sales'

Metrics:

- branch_id
- branch_name
- total_revenue
- transaction_count
- average_revenue_per_transaction

Formula:

total_revenue =
SUM(amount)

average_revenue_per_transaction =
SUM(amount) / COUNT(\*)

## Expense Metrics

Expense should be calculated from:

type = 'purchase'
type = 'expense'

Metrics:

- total_expenses

Formula:

SUM(amount)

## Net Profit Metrics

net_profit =
total_revenue - total_expenses

## Contribution Analysis

Calculate percentage contribution of each branch.

Formula:

(total_revenue / company_total_revenue) \* 100

Return:

[
{
"branch_id": 1,
"branch_name": "Kampala Branch",
"total_revenue": 12000000,
"total_expenses": 5000000,
"net_profit": 7000000,
"transaction_count": 120,
"average_revenue_per_transaction": 100000,
"revenue_contribution_percentage": 45
}
]

Sort:

ORDER BY total_revenue DESC

Dashboard Summary:

- Best Performing Branch
- Worst Performing Branch
- Total Company Revenue
- Total Company Expenses
- Total Company Profit
- Branch Contribution Breakdown

Implementation Notes:

- Use CashFlow as the primary source.
- Join BusinessBranch to retrieve branch names.
- Do not recalculate revenue from Sale or SaleItem.
- Use database aggregation functions (SUM, COUNT).
- Avoid N+1 queries.

# Dead Stock Report

Controller:
DeadStockReportController

Model:
BusinessBranchProduct

Purpose:
Identify products not sold recently.

Logic:

last_sold_at IS NULL
OR
last_sold_at <= now()->subDays(90)

Metrics:

- dead_stock_count
- dead_stock_value

dead_stock_value:

SUM(quantity \* cost_price)

Return:

- id
- name
- quantity
- cost_price
- last_sold_at
